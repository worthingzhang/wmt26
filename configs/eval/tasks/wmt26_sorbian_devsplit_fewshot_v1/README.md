# WMT26 Sorbian Devsplit Few-Shot Overlay Tasks

This directory is a **local overlay task directory** for the `devsplit_fewshot_v1` evaluation profile.

## What it is

The YAML files here define lm-eval tasks that run against the dev-split data generated by the project's devsplit builder. They are **overlay tasks** -- they do not modify or replace the official YAMLs located in `repos/official_eval`.

## Why `utils.py` symlinks exist in every subdirectory

lm-eval resolves `!function utils.X` **relative to each YAML file's directory**. Because the overlay tasks live in subdirectories (`qa/`, `mt/`, `sc/`, `gc/`, `mr/`), each subdirectory needs its own `utils.py` so that the YAMLs can reference shot-loading functions with `!function utils.hsbqa_shots`, `!function utils.de_hsb_mt_shots`, etc.

To avoid duplicating code, each subdirectory's `utils.py` is a **symlink** to the top-level `utils.py` in this directory.

## Two `utils.py` modules -- do not confuse them

1. **`utils.py` in this overlay directory** -- provides shot-loading helpers (`hsbqa_shots`, `de_hsb_mt_shots`, etc.) used by the `fewshot_config.samples` field in the overlay YAMLs.
2. **`lm_eval.tasks.wmt26-lrl.utils` inside `repos/official_eval`** -- a separate module used for `process_docs`, `process_results`, and the `chrf_pp` metric function. The overlay YAMLs reference it with `!function lm_eval.tasks.wmt26-lrl.utils.process_level_a1`, `!function lm_eval.tasks.wmt26-lrl.utils.process_sc_results`, etc.
